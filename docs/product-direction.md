# Product Direction

## Demo Framing

The first version can be framed around a concrete learner:

- the user's husband
- a Japanese qualification exam
- an AI-operated coaching company that manages the learner's roadmap

This gives the project a real operating context while keeping the external product story understandable in English.

## Core User Value

The product should not mainly sell "conversation with AI."

It should sell:

- reduced planning burden
- continuous roadmap management
- pass-likelihood visibility
- guided next actions
- visible milestones that make progress feel real

## Core Product Concept

The student should mainly do what the system tells them to do.

Planning, strategy, and coaching should be handled by the product as the operating layer of the company.

This means:

- the student studies
- the student reports results
- the system updates the roadmap
- the system decides the next step
- the system shows the current milestone and pass trajectory

## Interaction Style

The first interface should not overload the student with choices.

The ideal interaction is closer to a private tutoring session in a cafe:

- the coach starts the session
- the coach asks how much time is available
- the coach proposes a realistic goal for the session
- the student follows along
- the student can ask for current pass probability when needed

The interaction should feel supportive, but also directive.

The system should not keep returning control to the student with broad choices.
It should lead the student forward with the next sensible action.

Example interaction:

- Coach: Good morning. Shall we begin?
- Coach: How much time do you have today?
- Coach: Based on your current status, we should aim to solve three civil law questions today.
- Coach: Can you do 30 minutes?
- Student: What is my current pass probability?
- Coach: You are at 60 percent right now. If you continue at the current pace until December, that can rise to 80 percent.

## Interface Implication

The product should move away from a pure chatbox and toward guided operation.

Possible visible sections can still exist, but they should support a teacher-led flow:

- current session goal
- today's focus
- current milestone
- pass probability
- quick progress reporting

The system should lead. The student should respond.

Visible milestones matter because they help motivation.

When the learner can see where they are on the path, what is next, and how that affects pass probability, effort becomes easier to sustain.

## Feature Priority

1. The roadmap works end to end.
2. The coach can start a study session and guide the student.
3. Progress reports update the plan.
4. Pass probability is shown with a simple rationale.
5. Voice or avatar is optional polish if time remains.
