# Company Onboarding

## Core Principle

When a learner joins the Coaching Company, the system should not simply continue the LINE experience unchanged.

The Coaching Company should start with its own onboarding process.

## Why A Separate Onboarding Is Needed

The Coaching Company is a different product tier.

It is not just a new interface over the same daily drill stream.

It should begin by establishing:

- current level
- likely weak areas
- available study capacity
- immediate roadmap direction

This creates the feeling that the learner is now being actively managed by the company.

## Recommended First Steps

1. Initial greeting and setup
2. Current status confirmation
3. Short assessment or diagnostic quiz
4. Creation of Coaching Company state
5. First roadmap and first guided session

## Relationship To Existing LINE Data

Existing LINE data can be used as reference input.

Examples:

- past activity volume
- streak behavior
- previously seen topics
- prior question exposure

However, Coaching Company should not blindly inherit all LINE state.

Instead:

- LINE data informs the starting estimate
- Company onboarding creates a new managed state

## State Boundary

When the learner joins the Coaching Company, the system should create a new product-specific state for that learner.

This state should include:

- current subject
- current milestone
- risk level
- pass probability
- next action today

This is the company's live operating state, not just a continuation of LINE progress.

## Assessment Use

The first Company session should ideally include a short check such as:

- current status review
- practical diagnostic questions
- targeted weakness confirmation

This allows the AI company to say:

"First, let us confirm where you are now."

That is a much stronger premium experience than simply continuing the exact same drill stream.

## Question Policy During Onboarding

The onboarding assessment should avoid careless duplication from LINE.

It may:

- use a separate diagnostic set
- use selected shared questions with assessment purpose
- reuse prior topics only when intentionally validating weakness

The important point is that the learner feels evaluated, not recycled.
